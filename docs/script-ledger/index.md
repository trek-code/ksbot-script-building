## Status Guide

- `Planned`: design or research exists, implementation not started
- `In Progress`: active local development
- `Testing`: currently being transferred or validated in KSBot
- `Stable`: accepted working baseline
- `Archived`: no longer maintained

## Update Process

1. Update the script's history file in `scripts/`.
2. Add a version-specific release note in `releases/`.
3. Update this index with the newest version, date, and status.
4. Add or update the GitHub link if a repo path or release reference exists.
5. Reference the same version in handoff and test-run notes.
